7-Week Timeline
Phase 1
P1

Foundations

April 27 – May 10, 2026 · Weeks 1–2 · Platform: DataCamp

Stick to DataCamp only this phase. Build your base before you go wide. Goal: write Python and SQL from scratch without googling every line.

  • Python for Data Engineering track on DataCamp
  • SQL for Data Engineers — CTEs, window functions, aggregations, joins
  • Introduction to data types: CSV, JSON, APIs, databases
  • Basic file handling in Python: reading, writing, transforming
Phase 1 Projects — Do them in order. Each maps to a real skill.
P1-A CSV Cleaner Beginner
What you build A Python script that reads a messy CSV file, cleans it (fix nulls, wrong types, duplicates) and outputs a clean version.
Why it matters This is 80% of real DE work. Every pipeline starts with dirty data. If you can clean data reliably, you are already useful.
Data source Download any messy CSV from kaggle.com. Good options: Titanic dataset, World Happiness Report, or any government open data CSV.
Tools Python (pandas), VS Code or Jupyter, CSV files
You can open a CSV with unknown columns, spot at least 3 data quality issues, fix them in code, and save a clean output file.
P1-B API to Database Loader Beginner
What you build Call a public REST API, pull JSON data, and insert it into a local SQLite or PostgreSQL database using Python.
Why it matters APIs are how real-world data enters pipelines. Databases are where it lives. This bridges those two things.
Data source Open-Meteo API (free, no key: open-meteo.com) — pull hourly weather for your city. Or OpenWeatherMap free tier.
Tools Python (requests, sqlite3 or psycopg2), SQLite or PostgreSQL
You can run your script, see new rows appear in the database, and query them with SQL to verify the data is correct.
Phase 1 Checkpoint: If you can do P1-A and P1-B without following a tutorial step-by-step, you are ready for Phase 2.
Phase 2
P2

Core Data Engineering Skills

May 11 – June 1, 2026 · Weeks 3–5 · Platform: O'Reilly (primary) + DataCamp

This is where the real DE concepts land. Pipelines, orchestration, containerisation. Do not skip Docker — it trips up almost everyone and shows up everywhere. Your O'Reilly access is open — use it heavily in these 3 weeks.

  • Fundamentals of Data Engineering — Joe Reis & Matt Housley (O'Reilly). Read Chapters 1–6 minimum.
  • Data Pipelines Pocket Reference (O'Reilly) — fast, practical, directly applicable
  • Apache Airflow basics — DAGs, tasks, scheduling, operators
  • Docker basics — images, containers, volumes, docker-compose
  • Cloud storage fundamentals — AWS S3 or GCS, buckets, object storage
Phase 2 Projects — P2-C is a stretch goal if time is tight.
P2-A Scheduled Data Pipeline with Airflow Intermediate
What you build An Airflow DAG that runs on a schedule: pulls data from an API, transforms it, and writes it to a database or CSV.
Why it matters Orchestration is the core job of a Data Engineer. Knowing Airflow is on almost every DE job posting.
Data source Open-Meteo API (api.open-meteo.com, no key) for hourly weather. Or CoinGecko API (free) for crypto prices.
Tools Apache Airflow (via pip or Docker), Python, PostgreSQL or SQLite, Docker
Your DAG runs on schedule without you doing anything. You can pause it, trigger it manually, and see task logs in the Airflow UI.
P2-B Dockerised ETL Script Intermediate
What you build Take your P1-B API loader and package it inside a Docker container so it can run anywhere.
Why it matters Docker is the difference between a script that works on your laptop and one you can ship to a client.
Data source Same API from P1-B. The data source doesn't change — the packaging does.
Tools Docker Desktop (free), Python, your P1-B script, a Dockerfile
You can type "docker build" and "docker run" and your ETL runs inside a container. Someone else can run it on their machine with one command.
P2-C Multi-Source Data Integration Stretch Goal
What you build Pull data from two different sources (API + CSV), merge on a common key, and load into one database table.
Data source Source 1: World Bank API (api.worldbank.org) for country GDP. Source 2: Country metadata CSV from kaggle.com. Join on country code.
Tools Python (pandas), requests, SQLite or PostgreSQL
A single database table exists with merged columns from both sources. You can query it with SQL and get sensible results.
Phase 2 Checkpoint: Can you explain what a DAG is, why Docker matters, and draw the flow of your P2-A pipeline on paper?
Phase 3
P3

Portfolio Project (Capstone)

June 2 – June 15, 2026 · Weeks 6–7 · Platform: Both + GitHub + free tools

This phase is the most important. One solid end-to-end project beats ten certificates. You are building the thing you point to when someone asks "what have you built?" Note: O'Reilly access ends June 15 — finish your reading in Phases 1 & 2.

P3 Automated Pipeline with Dashboard Capstone
What you build A complete, automated pipeline: ingest from a real public API daily, store in a database, run transformations to generate summary metrics, and expose a simple dashboard or report. Host the code on GitHub.
Why it matters This demonstrates ingestion, storage, transformation, scheduling, and presentation — every layer of the stack. This is your proof of work for clients and employers.
Pick one data source Football/Soccer: api.football-data.org (free tier) · NASA: api.nasa.gov (free key) · Nigerian economic data: dataportal.opendataforafrica.org · Air quality: openaq.org (free, no key)
Tools Python, Apache Airflow (scheduler), PostgreSQL (storage), dbt or plain SQL (transforms), Streamlit or Metabase (dashboard), Docker, GitHub
A GitHub repo exists with a README. The pipeline runs daily without you touching it. A dashboard shows at least 3 meaningful metrics. Someone else can clone and run it.
P3-BONUS Data Quality Monitor If Time Allows
What you build Add automated data quality checks to your capstone pipeline. Alert you (email or log) when data looks wrong: missing values above a threshold, unexpected nulls, value ranges that seem off.
Tools Great Expectations (Python library) or custom checks. Notifications via smtplib or a free Slack webhook.
Your pipeline automatically detects and logs at least 2 data quality issues. You have a record of when checks passed and failed.
This project is your CV. When you apply for freelance work, send this GitHub link. Make the README clear enough for someone non-technical to understand.

Freelance Entry Strategy — After June 15

Once Your Capstone Is on GitHub

  • Search Upwork / Toptal for "ETL pipeline", "data migration", "data cleaning", "Airflow developer"
  • Small businesses need someone to move spreadsheet chaos into a real database — that is your first gig
  • Lead with your GitHub link in every application or pitch
  • Your 2020 image classifier is a bonus — pitch yourself as "a DE who knows what ML teams need from data"
  • After 2–3 small contracts, start applying to MLOps-adjacent roles where DE skills are required

The goal is not to learn everything. It is to build something real.
Every project above is a brick. By June 15, you will have a house.
