Data Engineering is the foundation. It is also the clearest path to remote and freelance work. Building it first sets you up for MLOps naturally — not as a detour, but as the next logical step.

Phase 1 — Foundations

March 15 – April 11, 2026  ·  Weeks 1–4  ·  Platform: DataCamp

Stick to DataCamp only this phase. Build your base before you go wide. The goal is to feel comfortable writing Python and SQL from scratch without googling every line.

What to Study

  • Python for Data Engineering track on DataCamp
  • SQL for Data Engineers — CTEs, window functions, aggregations, joins
  • Introduction to data types: CSV, JSON, APIs, databases
  • Basic file handling in Python: reading, writing, transforming

Phase 1 Projects

Do them in order. They get harder. Each one maps to a real skill.

P1-A CSV Cleaner Beginner
What you buildA Python script that reads a messy CSV file, cleans it (fix nulls, wrong types, duplicates) and outputs a clean version.
Why it mattersThis is 80% of real DE work. Every pipeline starts with dirty data. If you can clean data reliably, you are already useful.
Data sourceDownload any messy CSV from kaggle.com (search "dirty data" or "raw data"). Good options: Titanic dataset, World Happiness Report, or any government open data CSV.
ToolsPython (pandas), VS Code or Jupyter, CSV files
✓ You are done when: You can open a CSV with unknown columns, spot at least 3 data quality issues, fix them in code, and save a clean output file.
P1-B API to Database Loader Beginner
What you buildCall a public REST API, pull JSON data, and insert it into a local SQLite or PostgreSQL database using Python.
Why it mattersAPIs are how real-world data enters pipelines. Databases are where it lives. This bridges those two things.
Data sourceOpen-Meteo API (free, no key needed: open-meteo.com) — pull hourly weather for your city. Or OpenWeatherMap (openweathermap.org) free tier with API key.
ToolsPython (requests, sqlite3 or psycopg2), SQLite (built into Python) or PostgreSQL (free install)
✓ You are done when: You can run your script, see new rows appear in the database, and query them with SQL to verify the data is correct.
Phase 1 Checkpoint: If you can do P1-A and P1-B without following a tutorial step-by-step, you are ready for Phase 2.

Phase 2 — Core Data Engineering Skills

April 12 – May 16, 2026  ·  Weeks 5–9  ·  Platform: O'Reilly (primary) + DataCamp

This is where the real DE concepts land. Pipelines, orchestration, containerisation. Do not skip Docker — it trips up almost everyone and shows up everywhere.

What to Study

  • Fundamentals of Data Engineering — Joe Reis & Matt Housley (O'Reilly book). Read Chapters 1–6 minimum. This gives you the mental model that makes everything else click.
  • Data Pipelines Pocket Reference (O'Reilly book) — fast, practical, directly applicable
  • Apache Airflow basics — DAGs, tasks, scheduling, operators
  • Docker basics — images, containers, volumes, docker-compose
  • Cloud storage fundamentals — pick AWS S3 or Google Cloud Storage, understand buckets and object storage

Phase 2 Projects

P2-A Scheduled Data Pipeline with Airflow Intermediate
What you buildAn Airflow DAG that runs on a schedule: pulls data from an API, transforms it (clean, filter, aggregate), and writes it to a database or CSV.
Why it mattersOrchestration is the core job of a Data Engineer. Knowing Airflow is on almost every DE job posting. This proves you can build something that runs automatically.
Data sourceOpen-Meteo API (api.open-meteo.com, no key needed) for hourly weather. Or CoinGecko API (coingecko.com/en/api, free, no key) for crypto prices.
ToolsApache Airflow (install via pip or Docker), Python, PostgreSQL or SQLite, Docker (recommended for Airflow setup)
✓ You are done when: Your DAG runs on schedule without you doing anything. You can pause it, trigger it manually, and see task logs in the Airflow UI.
P2-B Dockerised ETL Script Intermediate
What you buildTake your P1-B API loader and package it inside a Docker container so it can run anywhere, not just on your machine.
Why it mattersDocker is the difference between a script that works on your laptop and one you can ship to a client. This is a fundamental career skill.
Data sourceUse the same API from P1-B. The data source does not change — the packaging does.
ToolsDocker Desktop (free), Python, your existing script from P1-B, a Dockerfile
✓ You are done when: You can type "docker build" and "docker run" and your ETL runs inside a container. Someone else can run it on their machine with one command.
P2-C Multi-Source Data Integration Intermediate
What you buildPull data from two different sources (e.g. an API and a CSV file), merge them on a common key, and load the result into one database table.
Why it mattersReal DE work almost always involves multiple sources. Joining them cleanly is a core skill clients will pay for.
Data sourceSource 1: World Bank Open Data API (api.worldbank.org) for country GDP or population. Source 2: Download a country metadata CSV from kaggle.com or data.world. Join on country code.
ToolsPython (pandas), requests, SQLite or PostgreSQL, Jupyter Notebook for exploration
✓ You are done when: A single database table exists that has merged columns from both sources. You can query it with SQL and get sensible results.
Phase 2 Checkpoint: Can you explain what a DAG is, why Docker matters, and draw the flow of your P2-A pipeline on paper? If yes, you are Phase 3 ready.

Phase 3 — Portfolio Project (Capstone)

May 17 – June 13, 2026  ·  Weeks 10–13  ·  Platform: Both + GitHub + free tools

This phase is the most important. One solid end-to-end project beats ten certificates. You are building the thing you point to when someone asks "what have you built?"

P3 — CAPSTONE Automated Pipeline with Dashboard Capstone
What you buildA complete, automated pipeline: ingest from a real public API daily, store in a database, run transformations to generate summary metrics, and expose a simple dashboard or report. Host the code on GitHub.
Why it mattersThis demonstrates ingestion, storage, transformation, scheduling, and presentation — every layer of the stack. It is your proof of work for clients and employers.
Data source Pick one that interests you most:
• Football/Soccer stats: api.football-data.org (free tier)
• NASA open data: api.nasa.gov (free key at nasa.gov/open/data)
• Nigerian economic data: dataportal.opendataforafrica.org
• Air quality data: openaq.org (free, no key needed)
• Reddit trends: Reddit public API or pushshift.io
ToolsPython, Apache Airflow (scheduler), PostgreSQL (storage), dbt or plain SQL (transforms), Streamlit or Metabase (free dashboard), Docker, GitHub
✓ You are done when: A GitHub repo exists with a README explaining what it does. The pipeline runs daily without you touching it. A dashboard shows at least 3 meaningful metrics. Someone else can clone and run it.
This project is your CV. When you apply for freelance work or jobs, you send this GitHub link. Make the README clear enough that someone non-technical can understand what the project does and why.

Bonus Project (If You Have Time)

P3-BONUS Data Quality Monitor Intermediate
What you buildAdd automated data quality checks to your capstone pipeline. Alert you (email or log) when data looks wrong: missing values above a threshold, unexpected nulls, value ranges that seem off.
Why it mattersData quality is one of the first things senior engineers ask about. Adding this makes your capstone look production-grade, not just a learning project.
Data sourceSame data as your capstone. No new data source needed.
ToolsGreat Expectations (Python library, free) or simple custom checks in Python. Notifications via smtplib (email) or a free Slack webhook.
✓ You are done when: Your pipeline automatically detects and logs at least 2 data quality issues. You have a record of when checks passed and failed.

Freelance Entry Strategy — After June 13

Once your capstone is on GitHub, here is the realistic path to first paid work:

  • Upwork / Toptal: Search for "ETL pipeline", "data migration", "data cleaning", "Airflow developer"
  • Small businesses need someone to move spreadsheet chaos into a real database — that is your first gig
  • Lead with your GitHub link in every application or pitch
  • Your 2020 image classifier is a bonus — pitch yourself as "a DE who knows what ML teams need from data"
  • After 2–3 small contracts, start applying to MLOps-adjacent roles where DE skills are required

The goal is not to learn everything. It is to build something real.

Every project above is a brick. By June 13, you will have a house.
